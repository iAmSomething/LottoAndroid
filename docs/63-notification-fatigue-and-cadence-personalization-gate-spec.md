# 알림 피로도/빈도 개인화 게이트 스펙 (S34)

## 1. 목적
- 과도한 알림으로 인한 피로와 앱 이탈을 릴리즈 전에 차단한다.
- 사용자 맥락/활동 패턴에 맞춘 빈도 개인화로 알림 효율을 높인다.

## 2. 적용 범위
- 대상 채널: 주간 리마인드, 미확인 결과 알림, 상태 안내 알림
- 대상 컨텍스트: 최근 참여 빈도, 최근 클릭/무시 패턴, 야간 시간대, 워치 연동 상태
- 대상 화면: `Settings`, `Home`, `Wear Settings`

## 3. S34 Scorecard (N1~N5)
### N1. 알림 과다 억제
- 확인 항목
  - 동일 의미 알림이 단기간 중복 발송되지 않는다.
  - 사용자 무시 패턴이 누적되면 자동으로 빈도가 완화된다.
- PASS 기준
  - 스팸성 반복 알림 이슈 0건

### N2. 개인화 빈도 적합성
- 확인 항목
  - 최근 활동 패턴에 따라 알림 타이밍/빈도가 조정된다.
  - 개인화 규칙이 예측 가능하고 설정에서 확인 가능하다.
- PASS 기준
  - 개인화 규칙 불명확 이슈 0건

### N3. 조용한 시간/예외 존중
- 확인 항목
  - 수면/집중 시간대에 비긴급 알림이 억제된다.
  - 긴급성 높은 알림만 정책에 따라 예외 처리된다.
- PASS 기준
  - 조용한 시간 정책 위반 0건

### N4. 제어 가능성/복구 경로
- 확인 항목
  - 사용자가 알림 빈도/종류를 쉽게 조정하고 즉시 반영 결과를 확인할 수 있다.
  - 잘못된 개인화 결과를 1~2탭 내에서 복원할 수 있다.
- PASS 기준
  - 설정 반영 지연/복구 불가 이슈 0건

### N5. 접근성/성능/배터리 동등성
- 확인 항목
  - 접근성 조건에서도 알림 의미/우선순위가 동일하게 인지된다.
  - 빈도 개인화 로직이 배터리/성능에 과도한 부담을 주지 않는다.
- PASS 기준
  - 접근성/성능/배터리 회귀 0건

## 4. 개선 우선순위
1. 중복/과다 알림 억제 규칙 고정(N1)
2. 활동 기반 빈도 개인화 규칙 가시화(N2)
3. 조용한 시간과 예외 정책 정합성 확보(N3)
4. 사용자 제어/복구 동선 단순화(N4)
5. 접근성/성능/배터리 동등성 검증(N5)

## 5. 예외처리/안정성/성능 연동
- 네트워크/백엔드 지연
  - 마지막 유효 정책으로 안전 fallback하고 중복 발송 가드를 유지한다.
- 시간대/로캘 변화
  - 로컬 시간 변경 시 조용한 시간 규칙을 즉시 재평가한다.
- 저전력 모드
  - 비핵심 알림 빈도를 자동 완화해 배터리 소모를 억제한다.

## 6. 테스트/릴리즈 연동 규칙

  - 조용한 시간 억제/예외 처리 로그(최소 1건)
  - 설정 변경 즉시 반영/복구 로그(최소 1건)
  - `S34` scorecard 최종 판정표(PASS/WARN/FAIL)
- 파일 규칙
  - `docs/assets/visual-proof-matrix/` + `docs/assets/distribution/` 하위에 날짜 태그로 저장
