# 상호작용 신뢰 신호/실행 확인 게이트 스펙 (S28)

## 1. 목적
- "눌렀는데 실행됐는지 모르겠다"는 체감 문제를 릴리즈 전에 차단한다.
- 사용자 액션마다 즉시 반응, 확정 상태, 실패 복구를 일관된 신뢰 신호로 제공한다.

## 2. 적용 범위
- 대상 화면: `Home`, `Result`, `Generator`, `Manage`, `Settings`
- 대상 액션: 버튼 탭, 시트 적용, 저장/삭제, 외부 이동, 재시도
- 상태 구간: 입력 직후, 처리 중, 완료, 실패

## 3. S28 Scorecard (R1~R5)
### R1. 즉시 반응 신호
- 확인 항목
  - 탭 직후 시각 또는 촉각 반응이 즉시 발생한다.
  - 입력이 수신됐음을 100ms 이내에 인지할 수 있다.
- PASS 기준
  - 즉시 반응 누락 0건

### R2. 확정 상태 가시성
- 확인 항목
  - 완료 상태가 명확히 표시된다(문구/아이콘/상태 배지).
  - 동일 액션 재탭 필요 여부가 분명하다.
- PASS 기준
  - 완료/진행 상태 혼선 0건

### R3. 실패 복구 경로 명확성
- 확인 항목
  - 실패 시 원인 요약 + 다음 행동(재시도/대안)이 함께 제공된다.
  - fallback 동선이 2탭 이내로 도달 가능하다.
- PASS 기준
  - 복구 경로 누락 0건

### R4. 중복 입력 억제
- 확인 항목
  - 처리 중 중복 탭이 안전하게 억제된다.
  - 중복 입력으로 인한 상태 불일치가 발생하지 않는다.
- PASS 기준
  - 중복 입력으로 인한 결함 0건

### R5. 접근성 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서도 실행 확인 정보가 유지된다.
  - 상태 전환 의미가 접근성 조건에 따라 손실되지 않는다.
- PASS 기준
  - 접근성 조건별 실행 확인 회귀 0건

## 4. 개선 우선순위
1. 탭 직후 즉시 반응 + 완료 표시 일관화(R1, R2)
2. 실패 복구 동선(재시도/fallback) 고정(R3)
3. 처리 중 중복 입력 억제 규칙 적용(R4)
4. 접근성 조건 동등성 검증(R5)

## 5. 예외처리/안정성/성능 연동
- 네트워크 지연/실패
  - 진행 상태를 명시하고 타임아웃 시 복구 액션을 즉시 제공한다.
- 외부 이동 실패
  - 복사/브라우저 열기 fallback으로 즉시 전환한다.
- 저사양 장치
  - 신뢰 신호는 최소 모션으로도 의미 전달이 가능해야 한다.

  - 완료 확인 UI(문구/배지/아이콘) 캡처
  - 실패 복구(fallback/재시도) 재현 로그(최소 1건)
