# 세션 복귀/중단 복원 게이트 스펙 (S32)

## 1. 목적
- 전화 수신, 앱 전환, 프로세스 정리 이후에도 사용자 작업 맥락이 끊기지 않도록 릴리즈 전에 품질 게이트를 고정한다.
- 화면/입력/선택 상태 복원의 일관성을 확보해 재작업과 이탈을 줄인다.

## 2. 적용 범위
- 대상 화면: `Home`, `Result`, `Generator`, `Manage`, `Settings`
- 대상 이벤트: 백그라운드 전환, 외부 링크 이동 후 복귀, 프로세스 kill 후 재진입, 구성 변경
- 대상 상태: 필터/정렬/스크롤/폼 입력/선택 상태/임시 draft

## 3. S32 Scorecard (R1~R5)
### R1. 복귀 위치 정확성
- 확인 항목
  - 앱 복귀 시 사용자가 떠난 화면과 위치(탭/스크롤)가 유지된다.
  - 외부 이동 후 복귀 시 핵심 CTA와 문맥이 즉시 재노출된다.
- PASS 기준
  - 복귀 위치 이탈 이슈 0건

### R2. 입력/선택 상태 보존
- 확인 항목
  - 입력 중 폼, 선택값, 필터/정렬 상태가 복귀 후 유지된다.
  - 임시 draft가 손실되지 않고 사용자에게 명확히 표시된다.
- PASS 기준
  - 입력/선택 상태 유실 0건

### R3. 중단 후 재개 흐름 명확성
- 확인 항목
  - 중단 후 재개 시 "이전 작업 계속" 경로가 1탭 이내 제공된다.
  - 이미 완료된 액션이 중복 실행되지 않도록 가드된다.
- PASS 기준
  - 재개 동선 혼선/중복 실행 0건

### R4. 실패 복구/예외 안내
- 확인 항목
  - 복원 실패 시 원인 요약과 복구 액션(재시도/초기화/대체 경로)이 제공된다.
  - 복구 경로가 2탭 이내에 도달 가능하다.
- PASS 기준
  - 복원 실패 dead-end 0건

### R5. 접근성/성능 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서도 복귀 문맥이 동일하게 인지된다.
  - 복귀 시 첫 인터랙션 가능 시간과 화면 안정성이 임계치 이내를 유지한다.
- PASS 기준
  - 접근성/성능 조건별 복귀 회귀 0건

## 4. 개선 우선순위
1. 복귀 위치/탭/스크롤 복원 고정(R1)
2. 입력/선택/draft 상태 보존 규칙 고정(R2)
3. 재개 동선과 중복 실행 가드 표준화(R3)
4. 복원 실패 복구 경로 완결성 확보(R4)
5. 접근성/성능 동등성 검증(R5)

## 5. 예외처리/안정성/성능 연동
- 프로세스 kill/메모리 압박
  - 최소 복원 스냅샷을 저장하고 재진입 시 단계적 복원을 적용한다.
- 외부 이동 장기 이탈 후 복귀
  - 세션 타임스탬프 기반으로 stale 여부를 안내하고 이어하기/새로고침 선택지를 제공한다.
- 저사양 장치
  - 복귀 직후 무거운 계산은 지연 실행하고 사용자 상호작용을 우선 가능 상태로 만든다.
