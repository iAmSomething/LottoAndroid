# 멀티디바이스 상태 동기화/충돌 복구 게이트 스펙 (S33)

## 1. 목적
- 폰/워치/위젯 등 다중 진입점에서 상태 불일치로 인한 혼선과 재작업을 릴리즈 전에 차단한다.
- 동시 수정, 지연 동기화, 오프라인 재접속 상황에서 사용자 신뢰를 유지한다.

## 2. 적용 범위
- 대상 진입점: `Phone App`, `Wear App`, `Widget`, `Notification Action`
- 대상 데이터: 최근 회차, 번호 조합 상태, 보관/삭제 상태, 설정 토글, 마지막 조회 컨텍스트
- 대상 이벤트: 동시 편집, 오프라인 변경 후 재접속, 백그라운드 동기화, 실패 재시도

## 3. S33 Scorecard (M1~M5)
### M1. 상태 일관성
- 확인 항목
  - 동일 엔티티가 폰/워치/위젯에서 동일 상태로 보인다.
  - 최종 상태 반영 지연이 허용 범위 내(정의된 SLA)로 유지된다.
- PASS 기준
  - 관측 가능한 상태 불일치 0건

### M2. 충돌 감지/해결 명확성
- 확인 항목
  - 동시 수정 충돌이 감지되고 해결 규칙(last-write-wins/사용자 선택)이 명시된다.
  - 사용자에게 충돌 해결 결과와 영향 범위가 이해 가능하게 안내된다.
- PASS 기준
  - 충돌 무음 덮어쓰기 이슈 0건

### M3. 오프라인 복원력
- 확인 항목
  - 오프라인 변경이 로컬 큐에 안전 저장되고 재접속 시 순차 반영된다.
  - 반영 실패 항목은 재시도 경로와 함께 노출된다.
- PASS 기준
  - 오프라인 변경 유실 0건

### M4. 재시도/복구 경로 완결성
- 확인 항목
  - 동기화 실패 시 자동 재시도와 수동 재시도 경로가 모두 제공된다.
  - fallback 경로(수동 동기화/새로고침)가 2탭 이내 도달 가능하다.
- PASS 기준
  - 동기화 실패 dead-end 0건

### M5. 접근성/성능 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서도 동기화 상태 의미가 유지된다.
  - 동기화 UI 표시가 과도한 렌더링 비용 없이 반응성을 유지한다.
- PASS 기준
  - 접근성/성능 조건별 동기화 회귀 0건

## 4. 개선 우선순위
1. 폰-워치-위젯 상태 일관성 보장(M1)
2. 충돌 감지/해결 규칙 가시화(M2)
3. 오프라인 큐/재접속 복원 경로 고정(M3)
4. 실패 재시도/수동 복구 동선 완결성 확보(M4)
5. 접근성/성능 동등성 검증(M5)

## 5. 예외처리/안정성/성능 연동
- 동시 수정 충돌
  - 충돌 이벤트를 기록하고 deterministic 규칙으로 해결한 뒤 사용자에게 요약한다.
- 장시간 오프라인
  - 변경 큐를 보존하고 재접속 시 배치 반영 + 실패 항목 재시도 리스트를 제공한다.
- 저사양 장치/워치
  - 동기화 상태 배지 업데이트를 경량화하고 의미 전달을 우선한다.
