# 매주로또 Product Overview

## 1. 제품 목표
- 매주 로또를 구매하는 사용자가 번호 준비, 구매 등록, 당첨 확인을 빠르게 끝낼 수 있도록 돕는다.
- 사용 빈도를 높이기 위해 홈 위젯과 주간 알림을 핵심 리텐션 장치로 사용한다.
- 서버 없이 기기 로컬 저장 기반으로 초기 운영 비용을 최소화한다.

## 2. 타겟 사용자
- 매주 정기 구매자
- 종이 복권을 구매하고 당첨 확인을 자주 까먹는 사용자
- 번호 생성/보관/비교를 한 앱에서 해결하고 싶은 사용자

## 3. 핵심 가치
- 스마트 번호 생성: 자동/수동/반자동 + 잠금 번호 유지
- 간편 등록: QR 스캔 및 직접 저장
- 빠른 결과 확인: 당첨 번호 자동 연동 + 자동 채점
- 홈 화면 접근성: 위젯 A/B로 앱 실행 없이 핵심 정보 확인

## 4. 1차 범위
- 홈, 번호생성, 당첨결과, 통계, 알림설정 화면
- QR 파싱 파이프라인(2가지 포맷 지원)
- 당첨 API 연동/캐시/자동 채점
- WorkManager 기반 구매/결과 알림
- Glance 위젯 타입 A/B

## 5. 비기능 요구
- minSdk 26, Kotlin/Compose 기반
- 앱 시작 후 3초 이내 홈 진입
- 오프라인 시 마지막 캐시 결과 표시
- 단위 테스트(핵심 로직) + CI 빌드/정적분석

## 6. 2차 고도화 목표 (2026-Q2)
- 루틴 강화: 저장 후 결과 확인까지의 주간 루틴 완료율 개선
- 인사이트 강화: 통계를 행동 유도형 정보로 확장(주간 요약/추세)
- 접점 확장: Wear OS(갤럭시 워치)에서 핵심 정보 즉시 확인
- 운영 안정화: 장애 탐지/복구 시간 단축, 릴리즈 위험 사전 차단

## 7. 핵심 지표 (North Star + Supporting)
- North Star: 주간 루틴 완료 사용자 수(번호 저장 + 결과 확인)
- Supporting KPI
  - WAU
  - 4주 리텐션
  - 알림 클릭률
  - 결과 화면 진입률
  - 크래시 없는 세션 비율

## 8. 확장 원칙
- "기능 추가"보다 "사용자 행동 변화"를 우선한다.
- 큰 기능은 2주 실험 단위로 검증 후 정식 반영한다.
- 플랫폼 확장(Wear OS)은 읽기 중심 기능부터 단계적으로 적용한다.
