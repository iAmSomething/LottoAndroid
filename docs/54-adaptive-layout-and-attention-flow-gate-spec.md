# 적응형 레이아웃/시선 흐름 게이트 스펙 (S25)

## 1. 목적
- 화면 크기(소형/표준/대형)에 따라 정보 밀도와 CTA 발견성이 흔들리는 문제를 릴리즈 전에 차단한다.
- 동일 기능이 어떤 화면 크기에서도 같은 우선순위와 스캔 흐름으로 인지되도록 고정한다.

## 2. 적용 범위
- 핵심 화면: `Home`, `Result`, `Generator`, `Manage`, `Settings`
- 디바이스 구간
  - 소형: 폭 360dp 이하
  - 표준: 폭 361~411dp
  - 대형: 폭 412dp 이상
- 접근성 조건: 폰트스케일 1.0x/1.3x

## 3. S25 Scorecard (L1~L5)
### L1. 첫 시선 도달성
- 확인 항목
  - 핵심 CTA와 핵심 숫자가 첫 스크린에서 동시에 인지 가능하다.
  - 화면 크기가 바뀌어도 첫 시선 도달 순서가 유지된다.
- PASS 기준
  - 첫 시선 우선순위 역전 이슈 0건

### L2. 레이아웃 밀도 일관성
- 확인 항목
  - 카드 간 간격/섹션 간격/텍스트 블록 밀도가 구간별로 과밀/과소되지 않는다.
  - 정보량이 많은 화면(Result/Manage)에서 스캔 흐름이 끊기지 않는다.
- PASS 기준
  - 과밀/과소 레이아웃 이슈 0건

### L3. 엄지 영역 접근성
- 확인 항목
  - 주요 액션이 한 손 사용 기준 하단 접근 영역에 배치된다.
  - 소형 화면에서 핵심 액션이 상단 고립되지 않는다.
- PASS 기준
  - 핵심 액션 접근성 이슈 0건

### L4. CTA 우선순위 유지
- 확인 항목
  - 화면당 핵심 CTA 1개 원칙이 구간별로 유지된다.
  - 보조 액션이 핵심 CTA를 시각적으로 역전하지 않는다.
- PASS 기준
  - CTA 우선순위 충돌 0건

### L5. 폰트스케일 안정성
- 확인 항목
  - 1.3x 폰트스케일에서 줄바꿈/잘림/겹침 없이 핵심 정보가 유지된다.
  - 상태 카드와 버튼 라벨의 의미 전달이 손상되지 않는다.
- PASS 기준
  - 폰트스케일 관련 레이아웃 회귀 0건

## 4. 개선 우선순위
1. 소형 구간: CTA 접근성/밀도 보정(L1, L3)
2. 표준 구간: 기본 스캔 리듬과 우선순위 고정(L2, L4)
3. 대형 구간: 공백 과다/정보 분산 방지(L2, L4)
4. 1.3x 폰트스케일 전 구간 안정화(L5)

## 5. 예외처리/안정성/성능 연동
- 저사양 장치
  - 레이아웃 계산 비용이 증가하는 경우 장식 요소를 우선 축소한다.
- 네트워크/오류 상태
  - 상태 카드의 핵심 안내/행동 버튼이 레이아웃 변화에도 우선 노출된다.
- 외부 이동/복귀
  - 복귀 후에도 이전 시선 흐름과 CTA 위치가 크게 바뀌지 않도록 유지한다.
