# 권한 프릭션/점진적 동의 게이트 스펙 (S35)

## 1. 목적
- 권한 요청 시점/문맥이 부적절해 이탈이 발생하는 문제를 릴리즈 전에 차단한다.
- 사용자 신뢰를 유지하면서 필요한 권한만 단계적으로 요청하도록 UX 기준을 고정한다.

## 2. 적용 범위
- 대상 권한: 알림, 카메라(QR), 저장소/미디어(공유), 백그라운드 동작 관련 권한
- 대상 화면: `Home`, `Generator`, `Result`, `Settings`, `QR Flow`, `Wear Handoff`
- 대상 이벤트: 최초 진입, 기능 사용 직전, 권한 거부 후 재시도, 설정 복귀

## 3. S35 Scorecard (C1~C5)
### C1. 맥락 기반 요청 타이밍
- 확인 항목
  - 권한은 기능 사용 직전 맥락에서 요청된다(앱 시작 즉시 일괄 요청 금지).
  - 요청 전 "왜 필요한지"가 1문장으로 명확히 안내된다.
- PASS 기준
  - 무맥락 권한 요청 0건

### C2. 점진적 동의 흐름
- 확인 항목
  - 필수/선택 권한이 구분되어 단계적으로 요청된다.
  - 선택 권한 거부 시 핵심 기능이 불필요하게 차단되지 않는다.
- PASS 기준
  - 권한 의존도 과잉 결합 이슈 0건

### C3. 거부/재요청 복구 경로
- 확인 항목
  - 거부 후 대체 경로와 재요청 동선이 2탭 이내 제공된다.
  - 영구 거부 상태에서는 시스템 설정 이동 경로가 명확하다.
- PASS 기준
  - 권한 거부 후 dead-end 0건

### C4. 마이크로카피/신뢰 신호
- 확인 항목
  - 권한 안내 문구가 과장/불안 유발 없이 기능 가치 중심으로 작성된다.
  - 허용/거부 결과가 사용자에게 즉시 피드백된다.
- PASS 기준
  - 권한 문구 오해/불신 유발 이슈 0건

### C5. 접근성/성능/배터리 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서도 권한 흐름 의미가 유지된다.
  - 권한 상태 확인 로직이 과도한 polling 없이 동작한다.
- PASS 기준
  - 접근성/성능/배터리 회귀 0건

## 4. 개선 우선순위
1. 맥락 기반 요청 타이밍 고정(C1)
2. 필수/선택 권한 분리와 점진적 동의 체계 고정(C2)
3. 거부 후 복구 동선 완결성 확보(C3)
4. 권한 마이크로카피 신뢰도 개선(C4)
5. 접근성/성능/배터리 동등성 검증(C5)

## 5. 예외처리/안정성/성능 연동
- OS 버전별 권한 정책 차이
  - 버전 분기 테이블을 유지하고 지원 불가 권한은 graceful fallback 처리한다.
- 권한 상태 캐시 불일치
  - 앱 복귀 시 권한 상태를 재동기화하고 stale 상태를 정리한다.
- 저전력/백그라운드 제한
  - 권한 상태 확인 주기를 조정해 배터리 소모를 억제한다.
