# 장애 대응 루틴 (Runbook)

## 1. 목적
- 배포 이후 앱 장애 발생 시 재현, 영향도 파악, 임시완화, 패치 배포까지의 표준 절차를 정의한다.

## 2. 우선순위 분류
- P0: 앱 실행 불가, 데이터 손실, 결제/개인정보 치명 이슈
- P1: 핵심 기능(번호저장/QR/결과조회/알림) 불가
- P2: 일부 화면/기능 비정상, 우회 가능
- P3: 경미한 UI/문구/로그 이슈

## 3. 대응 타임라인
1. 장애 접수 후 10분 내 등급(P0~P3) 지정
2. 30분 내 재현 여부/영향 범위 공유
3. 60분 내 임시 완화책(안내문/기능 우회/롤백) 결정
4. 당일 내 핫픽스 필요 여부 판단 및 배포 계획 수립

## 4. 실무 체크리스트
- [ ] 재현 단말/OS/API 레벨 기록
- [ ] 로그 수집(`adb logcat`, 앱 내부 로그)
- [ ] 서버 연동 이슈 시 네트워크 상태/응답 코드 확인
- [ ] 로컬 DB/캐시 영향 범위 점검
- [ ] 사용자 공지 문구 작성(필요 시)

## 5. 로컬 재현 명령
```bash
./gradlew :app:testDebugUnitTest
./gradlew :app:connectedDebugAndroidTest
./gradlew :app:assembleRelease
```

## 6. 커뮤니케이션 템플릿
- 현상: [무엇이 안 되는지]
- 영향: [사용자/기능 범위]
- 현재상태: [재현 여부, 우회 가능 여부]
- 다음액션: [담당자, ETA]

## 7. 종료 기준
- 재현 케이스 기준 수정 확인
- 회귀 테스트 통과
- 원인/조치/재발방지 항목을 `11-progress-tracker.md`에 기록
