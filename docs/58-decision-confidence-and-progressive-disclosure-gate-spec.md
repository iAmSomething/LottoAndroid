# 의사결정 신뢰도/점진적 공개 UX 게이트 스펙 (S29)

## 1. 목적
- 정보가 많아도 사용자가 핵심 결정을 빠르고 정확하게 내리도록 릴리즈 전에 품질 게이트를 고정한다.
- 요약 우선, 세부 점진 공개, 위험 정보 노출 타이밍의 일관성을 확보해 오판/이탈을 줄인다.

## 2. 적용 범위
- 대상 화면: `Home`, `Result`, `Generator`, `Manage`, `Settings`
- 대상 흐름: 구매 유도, 결과 확인, 번호 생성/저장, 설정 변경, 외부 이동 전 안내
- 상태 구간: 첫 노출(요약), 세부 확장, 위험/주의 안내, 최종 결정 CTA

## 3. S29 Scorecard (D1~D5)
### D1. 핵심 요약 우선 노출
- 확인 항목
  - 첫 화면에서 핵심 판단 정보(무엇을 해야 하는지, 왜 필요한지)가 1뷰 내에 보인다.
  - 세부 설명 없이도 다음 행동을 시작할 수 있다.
- PASS 기준
  - 요약 부재/핵심 정보 탐색 실패 0건

### D2. 세부 정보 점진 공개
- 확인 항목
  - 세부 정보는 단계적으로 확장되며 기본 화면을 과도하게 점유하지 않는다.
  - 펼침/접힘 상태가 예측 가능하고 재방문 시 일관된다.
- PASS 기준
  - 세부 공개 구조 혼선 0건

### D3. 위험 신호 타이밍
- 확인 항목
  - 연령 제한/시간 제한/외부 이동 주의사항이 결정 직전 적절한 시점에 노출된다.
  - 위험 정보가 늦게 노출되어 되돌아가는 경우가 없다.
- PASS 기준
  - 위험 정보 타이밍 오류 0건

### D4. 결정 CTA 확신 문구
- 확인 항목
  - CTA 주변 문구가 행동 결과를 명확히 설명한다(예: 외부 공식 페이지 이동).
  - 실행 후 기대 결과/다음 단계가 사용자에게 분명하다.
- PASS 기준
  - CTA 의미 오해/재탭 혼선 0건

### D5. 접근성 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서 요약-세부 구조가 동일하게 인지된다.
  - 핵심 결정까지 탭 수/시간이 접근성 조건에서 과도하게 증가하지 않는다.
- PASS 기준
  - 접근성 조건별 의사결정 흐름 회귀 0건

## 4. 개선 우선순위
1. 요약 정보 우선 노출 고정(D1)
2. 세부 공개 단계/밀도 표준화(D2)
3. 위험 신호 노출 타이밍 정렬(D3)
4. CTA 확신 문구/후속 안내 통일(D4)
5. 접근성 동등성 검증(D5)

## 5. 예외처리/안정성/성능 연동
- 네트워크/데이터 지연
  - 요약 영역은 즉시 노출하고, 세부 영역은 skeleton/지연 로딩으로 분리한다.
- 외부 이동 실패
  - fallback(링크 복사/기본 브라우저)과 복귀 경로를 2탭 이내로 제공한다.
- 저사양 장치
  - 세부 공개 애니메이션은 축약 가능해야 하며 핵심 정보 전달을 우선한다.
