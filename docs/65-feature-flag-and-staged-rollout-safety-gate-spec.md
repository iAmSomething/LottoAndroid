# 실험 플래그/점진 롤아웃 안전 게이트 스펙 (S36)

## 1. 목적
- 실험 플래그와 점진 롤아웃 중 발생할 수 있는 회귀/편향/운영 사고를 릴리즈 전에 차단한다.
- 빠른 실험 속도와 안전한 롤백 가능성을 동시에 확보한다.

## 2. 적용 범위
- 대상: feature flag, A/B 실험군 분기, staged rollout 퍼센트 배포
- 대상 표면: `Home`, `Result`, `Generator`, `Manage`, `Settings`, `Wear Handoff`
- 대상 이벤트: 플래그 on/off, 비율 변경, 긴급 롤백, 실험 종료/정리

## 3. S36 Scorecard (F1~F5)
### F1. 분기 결정 일관성
- 확인 항목
  - 동일 사용자 세션에서 분기 결정이 일관되게 유지된다.
  - 실험군/대조군 판정 기준과 우선순위가 문서화되어 있다.
- PASS 기준
  - 분기 흔들림/비결정 상태 0건

### F2. 관측성/지표 안전성
- 확인 항목
  - 분기별 핵심 KPI와 에러 지표가 분리 수집된다.
  - 실험 로그 누락 없이 event schema가 유지된다.
- PASS 기준
  - 분기별 계측 누락 0건

### F3. 롤아웃/롤백 제어성
- 확인 항목
  - 단계별 배포(예: 5%->20%->50%->100%)와 롤백 트리거가 명확하다.
  - 롤백 명령 이후 사용자 영향이 정의된 시간 내 안정화된다.
- PASS 기준
  - 롤백 불능/지연 사고 0건

### F4. 실패 격리/대체 경로
- 확인 항목
  - 실험군 결함 발생 시 대조군 경로로 안전 fallback 가능하다.
  - 실험 기능 미적용 시에도 핵심 사용자 여정이 유지된다.
- PASS 기준
  - 실험 실패로 핵심 플로우 차단 0건

### F5. 접근성/성능 동등성
- 확인 항목
  - 실험군/대조군 모두 접근성(스크린리더/폰트스케일/Reduce Motion)을 만족한다.
  - 분기 로직이 startup/interaction 지연을 유의미하게 증가시키지 않는다.
- PASS 기준
  - 접근성/성능 편차 회귀 0건

## 4. 개선 우선순위
1. 분기 결정 안정성과 문서화 고정(F1)
2. 분기별 계측/대시보드 일관성 확보(F2)
3. 롤아웃/롤백 운영 트리거 표준화(F3)
4. 실패 격리/대체 경로 완결성 확보(F4)
5. 접근성/성능 동등성 검증(F5)

## 5. 예외처리/안정성/성능 연동
- 원격 설정 지연/불가용
  - 마지막 안정 플래그 셋으로 fallback하고 안전 모드 분기를 유지한다.
- 실험군 에러 급증
  - 임계치 초과 시 자동 rollback 후보로 승격하고 운영 채널에 경보를 발행한다.
- 저사양/워치 장치
  - 분기 로직 계산 비용을 최소화하고 캐시 정책으로 지연을 억제한다.
