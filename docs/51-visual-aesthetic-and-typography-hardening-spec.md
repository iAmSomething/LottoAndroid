# UI 미감/타이포 완성도 게이트 스펙 (S22)

## 1. 목적
- 사용자가 느끼는 "기본 앱 느낌", "안 예쁨" 피드백을 릴리즈 전에 체계적으로 차단한다.
- 타이포/색/공백/상호작용의 완성도를 정량 점검 가능한 운영 게이트로 고정한다.

## 2. 적용 범위
- 모바일 앱 핵심 화면 4종: `Home`, `Result`, `Generator`, `Manage`
- 보조 확인 화면: `Settings`(외부 이동/리다이렉트 신뢰 UX), `Splash`(초기 인상/리듬)
- Wear는 `20-wearos-integration-plan.md` 기준을 따르되, 본 게이트의 A1/A2/A3 원칙을 동일 적용한다.

## 3. S22 Scorecard (A1~A5)
### A1. 타이포 개성/위계
- 확인 항목
  - 헤드라인/본문/메타가 3단 위계로 즉시 구분된다.
  - 숫자(회차/당첨금/카운트) 강조 스타일이 화면 간 일관된다.
  - 기본 시스템 폰트 인상이 강하게 남지 않는다.
- PASS 기준
  - 화면 4종 모두 위계 오류 0건
  - 숫자 강조 규칙 위반 0건

### A2. 시각 대비/포커스
- 확인 항목
  - 화면당 핵심 CTA 1개가 명확히 강조된다.
  - 카드/배경/텍스트 대비가 저조도와 1.3x 폰트스케일에서 유지된다.
- PASS 기준
  - CTA 우선순위 혼선 0건
  - 대비 경고 0건

### A3. 공백 리듬/밀도
- 확인 항목
  - 카드 간 간격, 섹션 간 간격, 타이포 라인하이트가 화면 간 일관된다.
  - 정보 밀도가 높은 화면(Result/Manage)에서도 스캔 흐름이 끊기지 않는다.
- PASS 기준
  - 레이아웃 압축/과밀 이슈 0건
  - 섹션 리듬 불일치 이슈 1건 이하

### A4. 상호작용 강약/모션 품질
- 확인 항목
  - press/sheet/list 전환 피드백이 과하거나 부족하지 않다.
  - Reduce Motion ON에서 의미는 유지되고 과한 모션은 제거된다.
- PASS 기준
  - press feedback P95 100ms 이내
  - sheet open->stable P95 180ms 이내
  - Reduce Motion 회귀 0건

### A5. 외부 이동 신뢰 UX
- 확인 항목
  - 외부 구매/리다이렉트 시 사용자에게 목적지/주의사항이 명확히 전달된다.
  - 실패 시 fallback(링크 복사/기본 브라우저) 경로가 2탭 이내로 도달 가능하다.
- PASS 기준
  - 외부 이동 실패 복구 성공률 95% 이상
  - fallback 도달 탭 수 2 이하

## 4. 개선 우선순위(실행 순서)
1. Home/Result: 타이포 위계 + CTA 대비 고정(A1/A2)
2. Generator/Manage: 공백 리듬 + 상호작용 강약 정렬(A3/A4)
3. Settings/Result 외부 이동 경로: 신뢰 UX/실패 fallback 정렬(A5)
4. Splash: 첫 인상 리듬(진입/축약)과 내부 화면 톤 연결

## 5. 예외처리/안정성/성능 연동
- 저사양 장치
  - 애니메이션 과부하 감지 시 축약 모드 강제
  - 시각 장식 레이어(질감/블러)는 성능 저하 시 단계적으로 축소
- 접근성
  - 1.3x 폰트스케일에서 줄바꿈/잘림/겹침 0건
  - Reduce Motion ON에서도 동일 기능 완료 가능
- 네트워크/외부 앱 실패
  - 외부 이동 실패는 사용자 행동 가능 액션(복사/브라우저 열기)을 항상 제공

## 7. 증적 팩(Evidence Pack)
- 필수 산출물
  - 화면 4종 전/후 캡처(1.0x, 1.3x, 저조도)
  - 상호작용 녹화 3종(press/sheet/list)
  - 외부 이동 실패 fallback 재현 로그 1건 이상
  - `S22` scorecard 최종 판정표(PASS/WARN/FAIL)
- 파일 규칙
  - `docs/assets/visual-proof-matrix/` + `docs/assets/distribution/` 하위에 날짜 태그로 저장
  - 판정 리포트는 `s22_visual_aesthetic_gate_<yyyy-mm-dd>.md` 형식 사용
