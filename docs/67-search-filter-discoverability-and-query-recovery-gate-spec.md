# 검색/필터 발견성 및 질의 복원 게이트 스펙 (S38)

## 1. 목적
- 사용자가 원하는 회차/번호/기록을 빠르게 찾지 못해 이탈하는 문제를 릴리즈 전에 차단한다.
- 검색/필터 사용 흐름에서 상태 유실 없이 탐색을 지속할 수 있도록 UX 기준을 고정한다.

## 2. 적용 범위
- 대상 화면: `Result`, `Manage`, `Home history`, `Settings history`
- 대상 기능: 검색창, 정렬/필터 시트, 최근 검색, 저장된 필터 프리셋
- 대상 이벤트: 질의 입력, 필터 적용/해제, 화면 전환/복귀, 앱 재진입

## 3. S38 Scorecard (Q1~Q5)
### Q1. 발견성
- 확인 항목
  - 검색/필터 진입 지점이 첫 화면에서 명확히 인지된다.
  - 주요 필터(기간/상태/출처)가 1~2탭 이내 접근 가능하다.
- PASS 기준
  - 검색/필터 진입 실패 이슈 0건

### Q2. 질의 결과 적합성
- 확인 항목
  - 질의 결과가 사용자 의도와 일치하며 노이즈 결과가 과도하지 않다.
  - 빈 결과 시 대체 제안(추천 키워드/필터 리셋)이 제공된다.
- PASS 기준
  - 결과 부정확/빈 결과 방치 0건

### Q3. 필터 상태 복원성
- 확인 항목
  - 화면 이동/복귀 후 검색어와 필터 상태가 유지된다.
  - 앱 재진입 시 마지막 유효 질의를 재사용할 수 있다.
- PASS 기준
  - 질의/필터 상태 유실 0건

### Q4. 제어 가능성/해제 용이성
- 확인 항목
  - 적용된 필터가 시각적으로 명확히 표시되고 즉시 해제 가능하다.
  - 복잡한 조합 필터도 reset 경로가 1탭 내 제공된다.
- PASS 기준
  - 필터 해제 혼선 0건

### Q5. 접근성/성능 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서도 검색/필터 의미가 유지된다.
  - 검색 입력/필터 적용 후 결과 반응 시간이 임계치 이내다.
- PASS 기준
  - 접근성/성능 회귀 0건

## 4. 개선 우선순위
1. 검색/필터 진입 발견성 강화(Q1)
2. 질의 결과 적합성 및 빈 결과 대체 흐름 고정(Q2)
3. 질의/필터 상태 복원 정책 고정(Q3)
4. 적용/해제 제어 UX 단순화(Q4)
5. 접근성/성능 동등성 검증(Q5)

## 5. 예외처리/안정성/성능 연동
- 대량 데이터/저사양 환경
  - 점진 로딩과 debounce 전략으로 응답성을 유지한다.
- 네트워크/캐시 불일치
  - 마지막 안정 결과 fallback과 동기화 상태 안내를 제공한다.
- 입력 중단/앱 전환
  - 미완료 질의를 안전하게 저장하고 복귀 시 재개한다.
