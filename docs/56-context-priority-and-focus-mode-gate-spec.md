# 컨텍스트 우선순위/집중 모드 게이트 스펙 (S27)

## 1. 목적
- 정보가 많아질수록 핵심 행동 도달이 느려지는 문제를 릴리즈 전에 차단한다.
- 기본 모드와 집중 모드에서 사용자 주의 분산을 줄이고 핵심 정보 우선순위를 안정적으로 유지한다.

## 2. 적용 범위
- 대상 화면: `Home`, `Result`, `Generator`, `Manage`, `Settings`
- 모드 구분: 기본 모드, 집중 모드
- 대상 요소: 핵심 CTA, 보조 정보 카드, 배지/알림, 보조 액션 영역

## 3. S27 Scorecard (F1~F5)
### F1. 핵심 정보 우선순위
- 확인 항목
  - 화면별 핵심 정보 1순위가 명확히 드러난다.
  - 핵심 CTA가 보조 정보에 가려지지 않는다.
- PASS 기준
  - 우선순위 역전 이슈 0건

### F2. 집중 모드 노이즈 억제
- 확인 항목
  - 집중 모드에서 비핵심 배지/알림/장식 요소가 단계적으로 축소된다.
  - 핵심 동선을 방해하는 요소가 제거된다.
- PASS 기준
  - 집중 모드 노이즈 잔존 이슈 0건

### F3. CTA 재배치 일관성
- 확인 항목
  - 모드 전환 시 CTA 위치/의미가 예측 가능하게 유지된다.
  - 탭/시트/리스트 맥락에서 CTA 동작이 변하지 않는다.
- PASS 기준
  - CTA 위치/동작 혼선 0건

### F4. 복귀 흐름 안정성
- 확인 항목
  - 집중 모드 해제 후 기존 컨텍스트(필터/스크롤/선택 상태)가 복원된다.
  - 외부 이동 후 복귀 시 우선순위 레이아웃이 깨지지 않는다.
- PASS 기준
  - 복귀 후 상태 유실 0건

### F5. 접근성 동등성
- 확인 항목
  - 1.3x 폰트스케일/Reduce Motion에서도 집중 모드의 의미 전달이 유지된다.
  - 핵심 행동 도달 경로가 모드/접근성 조건에 따라 과도하게 늘어나지 않는다.
- PASS 기준
  - 접근성 조건별 동선 회귀 0건

## 4. 개선 우선순위
1. 집중 모드에서 비핵심 요소 축소 규칙 고정(F2)
2. 핵심 CTA 우선 노출/재배치 규칙 정렬(F1, F3)
3. 모드 해제/외부 복귀 시 상태 복원성 강화(F4)
4. 접근성 조건 동등성 검증(F5)

## 5. 예외처리/안정성/성능 연동
- 알림/배지 급증 상황
  - 집중 모드에서 우선순위 규칙에 따라 자동 억제한다.
- 외부 이동 실패/복귀
  - 복귀 시 핵심 CTA와 현재 컨텍스트를 우선 복원한다.
- 저사양 장치
  - 모드 전환 시 장식 요소를 축소해 반응성을 유지한다.

## 6. 테스트/릴리즈 연동 규칙

  - 기본/집중 모드 전후 캡처(핵심 화면 4종)
  - 알림/배지 억제 동작 로그(최소 1건)
  - 모드 해제/외부 복귀 상태 복원 로그(최소 1건)
  - `S27` scorecard 최종 판정표(PASS/WARN/FAIL)
