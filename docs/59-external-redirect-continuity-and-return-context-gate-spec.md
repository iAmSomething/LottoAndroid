# 외부 리다이렉트 연속성/복귀 컨텍스트 게이트 스펙 (S30)

## 1. 목적
- 외부 공식 구매 페이지로 이동하는 흐름을 더 편하고 안전하게 만들어 이탈/혼선을 줄인다.
- 외부 이동 실패/복귀 시에도 사용자 맥락이 유지되도록 UX 연속성을 릴리즈 전에 고정한다.

## 2. 적용 범위
- 대상 화면: `Home`, `Result`, `Settings`, `Manage`
- 대상 액션: 외부 링크 이동 CTA, 1회 안내 모달, fallback 액션(링크 복사/브라우저 열기), 앱 복귀
- 상태 구간: 이동 전 안내, 이동 시도, 실패 복구, 복귀 후 재진입

## 3. S30 Scorecard (X1~X5)
### X1. 이동 의도 명확성
- 확인 항목
  - CTA/보조 문구에서 "공식 사이트로 이동" 의도가 즉시 인지된다.
  - 연령/시간 제한 같은 핵심 조건이 이동 전 요약 제공된다.
- PASS 기준
  - 외부 이동 의도 오해 0건

### X2. 이동 전 마찰 최소화
- 확인 항목
  - 1회 안내 모달 정책이 과도한 반복 노출 없이 동작한다.
  - 사용자 결정(바로 이동/취소/자세히 보기)이 1탭 내 선택 가능하다.
- PASS 기준
  - 불필요한 안내 반복 노출 0건

### X3. 실패 복구 경로 완결성
- 확인 항목
  - 외부 앱 미설치/인텐트 실패/네트워크 문제에서도 fallback 액션이 즉시 제공된다.
  - 링크 복사/기본 브라우저 열기 경로가 2탭 이내 도달 가능하다.
- PASS 기준
  - 리다이렉트 실패 후 dead-end 0건

### X4. 복귀 컨텍스트 복원성
- 확인 항목
  - 외부 이동 후 앱 복귀 시 기존 화면/필터/스크롤/선택 상태가 유지된다.
  - 복귀 직후 다음 행동(예: 결과 확인, 번호 관리) CTA가 끊기지 않는다.
- PASS 기준
  - 복귀 후 상태 유실 0건

### X5. 접근성/성능/보안 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x/Reduce Motion에서도 이동/복귀 흐름 의미가 유지된다.
  - 허용 도메인 검증/스킴 검증이 고정되고, 탭 반응은 100ms 이내 인지 가능하다.
- PASS 기준
  - 접근성/보안/반응성 회귀 0건

## 4. 개선 우선순위
1. 이동 의도/조건 요약 문구 고정(X1)
2. 1회 안내 모달 반복 억제 및 선택지 단순화(X2)
3. 실패 fallback 경로 표준화(X3)
4. 외부 복귀 컨텍스트 복원 보장(X4)
5. 접근성/성능/보안 동등성 검증(X5)

## 5. 예외처리/안정성/성능 연동
- 외부 앱 실행 실패
  - 인텐트 실패 즉시 fallback 다이얼로그를 노출하고 링크 복사 상태를 명확히 피드백한다.
- 허용되지 않은 URL/스킴
  - allowlist 차단 + 안전 경고 + 대체 경로(공식 홈페이지 루트)로 유도한다.
- 백그라운드 복귀 지연
  - 복귀 시 컨텍스트 restore를 우선 적용하고 무거운 연산은 지연 실행한다.
