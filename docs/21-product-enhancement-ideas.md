# 제품 고도화 아이디어 보드 (v2)

## 1. 목적
- MVP 이후 사용자 리텐션, 사용 빈도, 품질 안정성을 동시에 끌어올릴 아이디어를 체계적으로 관리한다.
- 본 문서는 "아이디어 풀"이며, 실제 착수 우선순위는 `22-prioritization-matrix.md`를 기준으로 한다.

## 2. 고도화 방향
- 루틴 강화: 매주 반복 사용 동기를 강화한다.
- 의사결정 지원: 단순 번호 저장에서 "행동 추천"으로 확장한다.
- 플랫폼 확장: 모바일 중심에서 Wear OS까지 루틴 접점을 확장한다.
- 운영 안정화: 장애 감지/복구 시간과 회귀 리스크를 낮춘다.

## 3. 아이디어 풀
## A. 코어 사용성
- `IDEA-A01` 저장 흐름 2탭 단축(생성→저장 클릭 수 축소)
- `IDEA-A02` 번호관리 빠른 액션(상세 진입 없이 복사/이동/삭제)
- `IDEA-A03` 회차 전환 스마트 기본값(최근 조회 회차 기억) - 1차 구현 완료(2026-02-27, Result 초기 진입 시 최근 조회 회차 우선 복원)
- `IDEA-A04` QR 실패 유형별 즉시 가이드(조명/각도/거리)
- `IDEA-A05` 공식 구매 페이지 리다이렉트 CTA(앱 내 구매 안내→외부 이동 단축)
- `IDEA-A06` 외부 이동 1회 안내 모달(성인/시간 제한 + 이동 확인)

## B. 리텐션/습관화
- `IDEA-B01` 개인 루틴 캘린더(최근 8주 구매/확인 히스토리)
- `IDEA-B02` 미확인 결과 배지(당첨결과 미확인 주차 강조)
- `IDEA-B03` 주간 리포트 카드(구매/당첨/순이익 요약)
- `IDEA-B04` 목표 기반 알림(예: 토요일 18시 이전 미등록 시 리마인드)

## C. 데이터 인사이트
- `IDEA-C01` 번호 조합 중복도 경고(최근 N주 내 동일 조합 비율) - 1차 구현 완료(2026-02-26, 통계 화면 경고 카드 + 번호생성 CTA/계측 연계 + 로그 샘플 루틴 운영 스크립트)
- `IDEA-C02` 번호 분포 히트맵(구간 편중 가시화)
- `IDEA-C03` 출처별 성과 비교(자동/수동/QR)
- `IDEA-C04` 기간별 ROI 트렌드 차트(4주/8주/전체)

## D. 플랫폼 확장
- `IDEA-D01` Wear OS 홈/번호/결과 요약 화면
- `IDEA-D02` 폰 핸드오프(워치에서 QR 스캔 화면 바로 열기)
- `IDEA-D03` 워치 컴플리케이션(이번주 번호 존재 여부/결과 상태) - v1 구현 완료(2026-02-27, SHORT/LONG_TEXT complication data source + 앱 실행 탭 액션)
- `IDEA-D04` 워치 알림 액션(나중에 보기/폰에서 열기) - v1 구현 완료(2026-02-27, 알림 액션 `앱에서 열기`/`30분 뒤 다시 알림` + snooze 재스케줄링)

## E. 운영/품질
- `IDEA-E01` API 응답 품질 모니터링(지연/실패율) - 1차 계측 반영(2026-02-26, `ops_api_request`)
- `IDEA-E02` 로컬 데이터 무결성 점검 잡(스키마/중복/깨진 레코드)
- `IDEA-E03` 크래시/ANR 자동 분류 템플릿
- `IDEA-E04` 릴리즈 위험 점수(테스트 커버리지/변경량 기반) - v1 운영 반영 완료(2026-02-27, 스크립트 + PR/주간 워크플로우)

## F. 비즈니스/확장 실험
- `IDEA-F01` 프리미엄 후보: 고급 통계(커스텀 기간, 상세 분포) - v1 구현 완료(2026-02-27, Stats 커스텀 회차 범위 필터 + 적용 검증 메시지)
- `IDEA-F02` 프리미엄 후보: 클라우드 백업/기기 간 복원 - v1 완료(2026-02-27, 로컬 백업/복원 기반: Settings 백업 파일 생성/복원)
- `IDEA-F03` 추천 모델 후보: 사용자 선호 번호 패턴 기반 제안
- `IDEA-F04` 파트너십 후보: 복권점 위치/운영시간 연계 - v1 구현 완료(2026-02-27, Home `근처 판매점 찾기` CTA + 지도 링크 fallback)

## G. 모션/상호작용
- `IDEA-G01` 콜드/웜 스타트 분리형 스플래시 애니메이션
- `IDEA-G02` 홈 로딩 skeleton + 브리지 전환
- `IDEA-G03` BallChip 선택/잠금/적중 상태 전이 모션 통일
- `IDEA-G04` 탭 전환/시트 오픈/리스트 삭제 모션 시스템화
- `IDEA-G05` 상호작용 햅틱 패턴(성공/주의/실패) 표준화
- `IDEA-G06` 접근성 Reduce Motion 모드 도입

## H. 비주얼/타이포 리프레시
- `IDEA-H01` 헤드라인/본문/숫자용 폰트 페어 도입(기본폰트 인상 탈피)
- `IDEA-H02` 숫자 강조형 타입 스케일(회차/당첨금/합계) 재구성
- `IDEA-H03` 보라 중심 톤에서 딥틸/웜골드 기반 팔레트로 점진 전환
- `IDEA-H04` 핵심 컴포넌트 4종(AppBar/TicketCard/BottomBar/BallChip) 비주얼 통일
- `IDEA-H05` 화면별 CTA 대비 규칙 재정의(화면당 1개 강조)

## I. UI 비주얼 폴리시 2차
- `IDEA-I01` 실제 브랜드 폰트 리소스(`res/font`) 연동 및 fallback 체계 확정
- `IDEA-I02` 카드 레이어 체계(기본/강조/위험) 및 그림자 규칙 통일
- `IDEA-I03` 홈/결과 핵심 카드에 제한적 배경 질감(그라디언트/노이즈) 적용
- `IDEA-I04` 아이콘 스타일 규격화(스트로크/라운드/크기 통일)
- `IDEA-I05` 화면별 전후 비교 기반 시각 품질 리뷰 루틴 운영

## J. 시각 완성도 체감 개선
- `IDEA-J01` 타이포 시안 2안(A: 신뢰형, B: 에디토리얼형) 동시 제작 후 비교 선택
- `IDEA-J02` Home/Result의 숫자 블록 자간/행간 미세 튜닝 규칙 문서화
- `IDEA-J03` 스플래시 시그니처 모션 1안(브랜드 컬러/볼 모티프) + Reduce Motion 동등안
- `IDEA-J04` 카드 표면 질감 강도 3단계(0/1/2) 실험으로 과한 장식 방지
- `IDEA-J05` 아이콘 라인 두께/라운드 반경 QA 체크리스트(혼합 스타일 제거)
- `IDEA-J06` 시각 증적 매트릭스 운영 자동화(파일명 규칙/판정 템플릿 기반)
- `IDEA-J07` 스플래시 모션-햅틱 매핑 표준(기본/Reduce Motion 동일 의미 전달)
- `IDEA-J08` 승인 패키지 인벤토리 보드(산출물 경로/상태/담당자 단일 뷰)
- `IDEA-J09` 승인 패키지 `missing → ready` 전환 체크포인트(주기/담당/증적 규칙) 자동 리마인드
- `IDEA-J10` 승인 패키지 1차 증적 우선 전략(완전체 전 1건 `ready` 선확보로 리스크 조기 축소)

## 4. 제외/보류 기준
- 법적/정책 리스크가 높은 기능(도박 조장 표현 등)
- 운영비 대비 가치가 낮은 서버 기능
- 실험 없이 대규모 개발이 필요한 제안
