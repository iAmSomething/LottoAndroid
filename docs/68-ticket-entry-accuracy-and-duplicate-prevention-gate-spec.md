# 번호 기록 입력 정확성/중복 방지 게이트 스펙 (S39)

## 1. 목적
- 로또 구매 기록 입력 과정에서 오입력/중복 저장으로 인한 신뢰 저하를 릴리즈 전에 차단한다.
- 사용자가 기록을 빠르게 입력하면서도 저장 결과를 확신할 수 있도록 UX/안정성 기준을 고정한다.

## 2. 적용 범위
- 대상 화면: `Generator save`, `QR 수동 보정`, `Manage create/edit`, `Ticket detail edit`
- 대상 기능: 번호 입력/수정, 회차 선택, 출처 선택, 저장/수정/삭제, 중복 감지 경고
- 대상 이벤트: 빠른 연속 입력, 저장 중 백그라운드 전환, 동일 기록 재저장, 네트워크/스토리지 오류

## 3. S39 Scorecard (T1~T5)
### T1. 입력 명확성/오입력 예방
- 확인 항목
  - 번호/회차/출처 입력 규칙이 화면에서 즉시 이해 가능하다.
  - 범위 외 값, 포맷 오류, 필수값 누락이 저장 전에 차단된다.
- PASS 기준
  - 오입력 저장 이슈 0건

### T2. 중복 감지/병합 제안
- 확인 항목
  - 동일 회차/동일 번호 조합 중복 저장 시 경고와 선택지(취소/병합/새로 저장)가 제공된다.
  - 중복 감지 기준(완전 동일/부분 동일)이 사용자 문구와 일치한다.
- PASS 기준
  - 중복 저장 방치 이슈 0건

### T3. 저장 안정성/복원성
- 확인 항목
  - 저장 중 앱 전환/중단 후 재진입 시 입력 상태가 복원된다.
  - 저장 실패 시 임시 입력 초안이 유실되지 않는다.
- PASS 기준
  - 저장/복원 상태 유실 0건

### T4. 수정/복구 용이성
- 확인 항목
  - 저장 직후 undo 또는 즉시 수정 경로가 제공된다.
  - 오류 발생 시 재시도/수정 진입이 2탭 이내로 가능하다.
- PASS 기준
  - 복구 경로 혼선 이슈 0건

### T5. 접근성/성능 동등성
- 확인 항목
  - 스크린리더/폰트스케일 1.3x에서도 입력/경고/확정 상태 의미가 유지된다.
  - 대량 기록/연속 저장 시 입력 반응성과 저장 완료 피드백이 임계치 이내다.
- PASS 기준
  - 접근성/성능 회귀 0건

## 4. 개선 우선순위
1. 저장 전 오입력 차단과 인라인 가이드 강화(T1)
2. 중복 감지 및 병합 선택 UX 고정(T2)
3. 저장 중 중단/복귀 복원 정책 고정(T3)
4. 저장 후 즉시 복구/수정 동선 단순화(T4)
5. 접근성/저사양 동등성 검증(T5)

## 5. 예외처리/안정성/성능 연동
- 연속 탭/중복 요청
  - 저장 버튼 debouncing + idempotency key로 중복 생성 차단
- 저장 실패/부분 저장
  - 로컬 초안 유지 후 실패 원인별 복구 CTA(재시도/수정/임시저장) 제공
- 대량 기록/저사양 기기
  - 비동기 검증 분리와 리스트 증분 갱신으로 프레임 드랍 최소화
