# KPI 및 실험 계획 (2026-Q2)

## 1. 목적
- 고도화 아이디어를 "기능 출시"가 아니라 "지표 개선" 중심으로 검증한다.

## 2. 핵심 KPI
- 주간 활성 사용자(WAU)
- 4주 리텐션(설치 후 4주차 재방문율)
- 주간 루틴 완료율(번호 저장 + 결과 확인)
- 알림 클릭률(구매/결과)
- 결과 화면 진입률(저장 사용자 대비)
- 크래시 없는 세션 비율
- ANR 비율
- 콜드 스타트 P95
- 주요 화면 스크롤 jank 비율

## 3. KPI 목표치 (초안)
- WAU: +20%
- 4주 리텐션: +8%p
- 주간 루틴 완료율: +15%
- 알림 클릭률: +10%p
- 크래시 없는 세션: 99.5% 이상
- ANR 비율: 0.2% 이하
- 콜드 스타트 P95: 2.2s 이하
- 주요 화면 jank 비율: 3% 이하

## 4. 실험 카드
## EXP-01 미확인 결과 배지
- 가설: 미확인 상태를 홈에 강조하면 결과 확인 진입률이 증가한다.
- 대상: 최근 2주 내 티켓 저장 사용자
- 성공 기준: 결과 화면 진입률 +12% 이상

## EXP-02 주간 리포트 카드
- 가설: 주간 요약 카드는 주간 재방문을 늘린다.
- 대상: 알림 활성 사용자
- 성공 기준: WAU +6% 이상, 이탈률 증가 없음

## EXP-03 Wear 홈/결과 요약
- 가설: 워치 노출은 결과 확인 루틴 완료율을 높인다.
- 대상: Wear OS 연결 사용자
- 성공 기준: 루틴 완료율 +10% 이상

## EXP-04 번호관리 빠른 액션
- 가설: 상세 진입 없는 빠른 액션은 관리 화면 체류 시간을 줄이고 완료율을 올린다.
- 대상: 번호관리 월 3회 이상 사용자
- 성공 기준: 작업 완료까지 평균 탭 수 -20%

## EXP-05 스플래시 모션 최적화
- 가설: 콜드/웜 분리형 스플래시는 초기 이탈률을 낮춘다.
- 대상: 신규 설치 후 첫 3회 실행 사용자
- 성공 기준: 첫 실행 10초 이내 이탈률 -8% 이상

## EXP-06 상호작용 피드백 강화
- 가설: 버튼/볼/시트 피드백 모션은 작업 완료율을 높인다.
- 대상: 번호생성/번호관리 활성 사용자
- 성공 기준: 저장 완료율 +6% 이상, 오류 재시도율 감소

## EXP-07 예외 메시지 표준화
- 가설: 오류 타입별 안내/재시도 문구를 표준화하면 실패 이탈률이 감소한다.
- 대상: 결과 조회/저장 실패 경험 사용자
- 성공 기준: 실패 후 재시도율 +10% 이상, 즉시 이탈률 -8% 이상

## EXP-08 성능 예산 게이트
- 가설: 스타트업/jank/ANR 예산 게이트를 릴리즈 루틴에 통합하면 품질 회귀를 줄일 수 있다.
- 대상: 내부 QA/프리릴리즈 빌드
- 성공 기준: 릴리즈 직전 성능 회귀 이슈 0건

## EXP-09 사용자 여정 예외 리허설
- 가설: 주요 여정(J01~J06) 실패 리허설을 주간 운영하면 장애 재현/복구 시간이 단축된다.
- 대상: 내부 QA/프리릴리즈 빌드
- 성공 기준: 실패 상황 최초 인지~복구 가이드 제공 시간 30% 단축

## EXP-10 핫패스 성능 최적화
- 가설: Home/Result/Manage 핫패스 최적화를 우선 적용하면 체감 로딩/스크롤 품질이 개선된다.
- 대상: 내부 QA + 프리릴리즈 사용자 샘플
- 성공 기준: 콜드스타트 P95 2.2s 이하 유지 + jank 3% 이하 유지

## EXP-11 하드닝 게이트/롤백 판정 자동화
- 가설: 하드닝 변경의 게이트 판정/롤백 기준을 표준화하면 배포 의사결정 시간이 단축되고 회귀 릴리즈를 줄일 수 있다.
- 대상: 내부 QA/릴리즈 운영자
- 성공 기준: 릴리즈 판정 리드타임 30% 단축 + 임계치 초과 빌드의 배포 진행 0건

## EXP-12 성능 게이트 캘리브레이션
- 가설: 에뮬레이터/실기기 프로파일 분리 + 반복측정 기반 판정을 적용하면 성능 게이트 오탐이 감소한다.
- 대상: 내부 QA/프리릴리즈 빌드
- 성공 기준: 성능 게이트 오탐률 30% 감소 + 릴리즈 판정 재측정 횟수 감소

## EXP-13 성능 게이트 운영 템플릿
- 가설: 실행 커맨드/리포트/판정 트리를 템플릿으로 고정하면 성능 게이트 운영 편차가 줄고 판정 속도가 개선된다.
- 대상: 내부 QA/릴리즈 운영자
- 성공 기준: 리포트 형식 누락 0건 + 판정 회의 소요시간 20% 단축

## EXP-14 성능 게이트 증적 패키지
- 가설: 증적 패키지(E1~E5) 제출 템플릿을 고정하면 릴리즈 판단 근거 누락이 줄고 의사결정 재작업이 감소한다.
- 대상: 내부 QA/릴리즈 운영자
- 성공 기준: 증적 누락 0건 + 판정 보류 후 재회의 비율 30% 감소
