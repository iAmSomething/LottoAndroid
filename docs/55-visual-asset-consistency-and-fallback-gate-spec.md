# 시각 자산 일관성/로딩 fallback 게이트 스펙 (S26)

## 1. 목적
- 아이콘/이미지/배지 자산의 스타일 불일치와 로딩 실패로 인한 완성도 저하를 릴리즈 전에 차단한다.
- 자산 로딩 성공/실패/오프라인 상황에서 사용자 경험이 안정적으로 유지되도록 고정한다.

## 2. 적용 범위
- 대상 자산: 아이콘, 일러스트/이미지, 배지/썸네일, 외부 로드 시각 리소스
- 대상 화면: `Home`, `Result`, `Generator`, `Manage`, `Settings`
- 상태 조건: 온라인, 네트워크 지연, 로딩 실패, 오프라인 복구

## 3. S26 Scorecard (V1~V5)
### V1. 자산 스타일 일관성
- 확인 항목
  - 스트로크 두께/라운드/톤이 화면 간 일관된다.
  - 배지/아이콘의 시각 언어가 컴포넌트 위계와 충돌하지 않는다.
- PASS 기준
  - 스타일 혼합 이슈 0건

### V2. 로딩/실패 fallback 명확성
- 확인 항목
  - 로딩 중 skeleton/placeholder가 목적과 의미를 전달한다.
  - 실패 시 대체 자산 또는 텍스트 fallback이 즉시 제공된다.
- PASS 기준
  - 로딩 실패 후 공백 노출 0건

### V3. 대비/가독성
- 확인 항목
  - 저조도/고명도/폰트스케일 1.3x에서 자산과 텍스트 대비가 유지된다.
  - 상태 배지와 본문 정보가 시각적으로 충돌하지 않는다.
- PASS 기준
  - 대비 경고 0건

### V4. 성능 예산
- 확인 항목
  - 자산 로딩으로 인한 화면 진입 지연이 체감되지 않는다.
  - 대형 자산 사용 시 메모리/스크롤 품질 회귀가 없다.
- PASS 기준
  - 자산 로딩 관련 성능 회귀 0건

### V5. 오프라인 복원성
- 확인 항목
  - 오프라인에서도 핵심 의미 전달 자산이 fallback으로 유지된다.
  - 온라인 복귀 시 자산이 안정적으로 재동기화된다.
- PASS 기준
  - 오프라인 복원 실패 0건

## 4. 개선 우선순위
1. 실패 fallback 공백 제거(V2, V5)
2. 스타일 언어 통일(아이콘/배지/썸네일) 고정(V1)
3. 저조도/1.3x 가독성 점검(V3)
4. 대형 자산 로딩 성능 회귀 방지(V4)

## 5. 예외처리/안정성/성능 연동
- 네트워크 실패
  - 로딩 타임아웃 시 fallback 자산/문구를 즉시 노출한다.
- 오프라인
  - 캐시 자산 우선 표시 후 복귀 시 재동기화한다.
- 저사양 장치
  - 고해상도 자산은 단계적으로 축소 로딩하여 메모리 부담을 완화한다.
